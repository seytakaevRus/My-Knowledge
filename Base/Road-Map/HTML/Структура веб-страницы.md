## Секции документа

Страницы могут по структуре отличаться между собой, но в них зачастую присутствуют следующие элементы:
- `header` - элемент, который обычно представлен в виде большой полосы сверху с большим заголовком, логотипом и, возможно, слоганом. Обычно остается неизменным от одной веб-страницы к другой в пределах веб-сайта;
- `navigation bar` - элемент, в котором присутствуют ссылки на главные секции сайта, обычно представлен в виде кнопок меню, ссылок или табов. Как и `header` обычно остается постоянным на других страницах в пределах веб-сайта. Этот элемент может быть частью `header`, но лучше их разделять для большой доступности;
- `main content` - элемент, представленный как большая область в центре, который содержит большинство уникального контента на странице, например, видео, которое ты хочешь посмотреть, главная история, которую ты читаешь или карта, которую ты хочешь просмотреть;
- `sidebar` - элемент, где содержится второстепенная информация - ссылки, цитаты, реклама. Обычно это связано контекстно с тем, что содержится в `main content` (например, на странице новостной статьи боковая панель может содержать биографию автора или ссылки на связанные статьи);
- `footer` - элемент, который обычно представлен в виде полосы снизу страницы. Обычно содержит мелкий шрифт, уведомления об авторских правах или контактную информацию. Иногда используется также в целях `SEO`, предоставляя ссылки для быстрого доступа к популярному контенту.

## Семантические обертки

### `<main>` Главный контент

#### Предназначение

Элемент `<main>` представляет из себя доминирующее содержимое `<body>` документа. Содержимое элемента `<main>` должно быть уникальным для документа. Этот элемент не влияет на структуру документа и является чисто семантическим.

В документе не должно быть более одного элемента `<main>`.
#### Атрибуты

Элемент принимает только глобальные атрибуты.
#### Доступность

Использование этого элемента позволяет узнать людям с ограниченными возможностями, где находится основная информация сайта, и сразу же перейти к её изучению.
#### Пример

```html
<main>
  <h1>Apples</h1>
  <p>The apple is the pomaceous fruit of the apple tree.</p>

  <article>
    <h2>Red Delicious</h2>
    <p>
      These bright red apples are the most common found in many supermarkets.
    </p>
  </article>

  <article>
    <h2>Granny Smith</h2>
    <p>These juicy, green apples make a great filling for apple pies.</p>
    <p>…</p>
    <p>…</p>
  </article>
</main>
```

### `<article>` Статья

#### Предназначение

Элемент `<article>` представляет собой автономную композицию в документе, странице, приложении или сайте, которая предназначена для распространения или повторного использования. Примерами могут быть - сообщение на форму, статья в журнале или газете, запись в блоге, карточка продукта, интерактивный виджет  или любой другой независимый элемент контента. Элементов `<article>` может быть несколько на странице.

В каждом `<article>` должен быть заголовок `<h1> - <h6>`, как ребенок `<article>`.

Когда элемент `<article>` вложен, внутренний элемент представляет статью, связанную с внешним элементом. Например, комментарии к сообщению в блоге могут быть элементами `<article>`, вложенными в `<article>`, который представлен как сообщение в блоге.

Информация об авторе элемента `<article>` может быть предоставлена через элемент `<address>` но только к внешним элементам.

Дату и время публикации элемента `<article>` можно описать с помощью атрибута `datetime` элемента `<time>`.

#### Атрибуты

Элемент принимает только глобальные атрибуты.
#### Пример

```html
<article class="film_review">
  <h2>Jurassic Park</h2>
  <section class="main_review">
    <h3>Review</h3>
    <p>Dinos were great!</p>
  </section>
  <section class="user_reviews">
    <h3>User reviews</h3>
    <article class="user_review">
      <h4>Too scary!</h4>
      <p>Way too scary for me.</p>
      <footer>
        <p>
          Posted on
          <time datetime="2015-05-16 19:00">May 16</time>
          by Lisa.
        </p>
      </footer>
    </article>
    <article class="user_review">
      <h4>Love the dinos!</h4>
      <p>I agree, dinos are my favorite.</p>
      <footer>
        <p>
          Posted on
          <time datetime="2015-05-17 19:00">May 17</time>
          by Tom.
        </p>
      </footer>
    </article>
  </section>
  <footer>
    <p>
      Posted on
      <time datetime="2015-05-15 19:00">May 15</time>
      by Staff.
    </p>
  </footer>
</article>```

### `<aside>` Дополнительный контент 

#### Предназначение

Элемент `<aside>` должен содержать контент, содержимое которого лишь косвенно связано с основным содержимым документа (словари, биография автора, ссылки по теме и т.д.). Таких элементы часто представлены в виде боковых панелей или выносок.
#### Атрибуты

Элемент принимает только глобальные атрибуты.
#### Пример

Контент внутри `<aside>` лишь косвенно связано с основным содержанием статьи.

```html
<article>
  <p>
    The Disney movie <cite>The Little Mermaid</cite> was first released to
    theatres in 1989.
  </p>
  <aside>
    <p>The movie earned $87 million during its initial release.</p>
  </aside>
  <p>More info about the movie…</p>
</article>
```

![[Структура_веб-страницы_aside.png]]

### `<section>` Общий раздел

#### Предназначение

Элемент `<section>` представляет элемент, где обычно содержит часть страницы, которая представляет собой единую функциональность (мини-карту, набор заголовков, тему). Зачастую элемент `<section>` должен в качестве иметь заголовок.

Этот элемент должен использоваться тогда, когда нет более подходящего семантического элемента. К примеру, для навигационного меню нужно использовать элемент `<nav>`, а не `<section>`.

Также рассмотри следующие кейсы:
- Если содержимое элемента представляет собой  независимую единицу контента, который имеет смысл объединить в отдельный элемент (сообщение в блоге, комментарий в блоге или газетная статья), то `<article>` будет лучшим вариантом;
- Если содержимое представляет собой полезную дополнительную информацию, которая работает вместе с основным контентом, но не является его непосредственной частью (например, связанные ссылки или биография автора), то `<aside>` следует использовать.
- Если содержимое представляет собой основную область содержимого документа, использовать `<main>`;
- Если ты используешь элемент только в качестве оболочки для стиля, то следует использовать вместо него `<div>`.
#### Атрибуты

Элемент принимает только глобальные атрибуты.
#### Пример

```html
<section>
  <button class="reply">Reply</button>
  <button class="reply-all">Reply to all</button>
  <button class="fwd">Forward</button>
  <button class="del">Delete</button>
</section>
```

![[Структура_веб-страницы_section.png]]

### `<header>` Заголовок

#### Предназначение

Элемент `<header>` представляет из себя группу вводных или навигационных средств. Он может содержать некоторые элементы заголовка, логотип, форму поиска, имя автора и другие элементы. Обычно располагается в начале страницы, но может быть и частью `<article>` и `<section>`.
#### Атрибуты

Элемент принимает только глобальные атрибуты.
#### Пример

```html
<header>
  <h1>Main Page Title</h1>
  <img src="mdn-logo-sm.png" alt="MDN logo" />
</header>
```

![[Структура_веб-страницы_header.png]]

### `<footer>` Подвал

#### Предназначение

Элемент `<footer>` может содержать информацию об авторе, данные об авторских правах, навигацию. Обычно располагается в конце страницы, но может быть и частью `<article>` и `<section>`.
#### Атрибуты

Элемент принимает только глобальные атрибуты.
#### Пример

```html
<body>
  <h3>FIFA World Cup top goalscorers</h3>
  <ol>
    <li>Miroslav Klose, 16</li>
    <li>Ronaldo Nazário, 15</li>
    <li>Gerd Müller, 14</li>
  </ol>

  <footer>
    <small>
      Copyright © 2023 Football History Archives. All Rights Reserved.
    </small>
  </footer>
</body>
```

![[Структура_веб-страницы_footer.png]]

### `<nav>` Навигационная панель

#### Предназначение

Элемент `<nav>` представляет собой раздел страницы, где представлены навигационные ссылки, которые ведут на текущий документ, либо на другие документы. Типичными примерами навигации являются меню, оглавления.

Не обязательно, что все ссылки были представлены в `<nav>`, достаточно только основных, остальные можно разместить в `<footer>`.
#### Атрибуты

Элемент принимает только глобальные атрибуты.
#### Пример

```html
<nav>
  <h2>Navigation</h2>
  <p>
    You are on my home page. To the north lies <a href="/blog">my blog</a>, from
    whence the sounds of battle can be heard. To the east you can see a large
    mountain, upon which many <a href="/school">school papers</a> are littered.
    Far up this mountain you can spy a little figure who appears to be me,
    desperately scribbling a <a href="/school/thesis">thesis</a>.
  </p>
  <p>
    To the west are several exits. One fun-looking exit is labeled
    <a href="https://games.example.com/">"games"</a>. Another more
    boring-looking exit is labeled <a href="https://isp.example.net/">ISP™</a>.
  </p>
  <p>
    To the south lies a dark and dank <a href="/about">contacts page</a>.
    Cobwebs cover its disused entrance, and at one point you see a rat run
    quickly out of the page.
  </p>
</nav>
```

## Не семантические обертки

### `<div>` Блочный контент

#### Предназначение

Элемент `<div>` - общий контейнер для содержимого. Он не оказывает никакого влияния на контент или макет до тех пор, пока не будет стилизован с помощью CSS. Является `блочным` элементом. Использовать его следует тогда, когда никакой другой семантический элемент не подходит.
#### Атрибуты

Элемент принимает только глобальные атрибуты.
#### Пример

```html
<div>
  <p>
    Any kind of content here. Such as &lt;p&gt;, &lt;table&gt;. You name it!
  </p>
</div>
```

![[Структура_веб-страницы_div.png]]

### `<span>` Строчный контент

#### Предназначение

Элемент `<span>` - общий контейнер для содержимого, который из себя ничего не представляет. Похож на `<div>`, но является `строчным` элементом. Использовать его следует тогда, когда никакой другой семантический элемент не подходит.
#### Атрибуты

Элемент принимает только глобальные атрибуты.
#### Пример

```html
<p><span>Some text</span></p>
```

![[HTML-теги_span.png]]

## Разрывы строк и горизонтальные линии

### `<br>` Разрыв строки

#### Предназначение

Элемент `<br>` используется для переноса текста на новую строку. Это полезно при написании поэмы или адреса, где разделение на строки важно.
#### Атрибуты

Элемент принимает только глобальные атрибуты.
#### Плохие практики

Использовать `<br>` для создания отступов между параграфами, для этого лучше текст обернуть в `<p>` и использовать свойство `margin`. 
#### Пример

`<br>` можно использовать, когда нужно несколько таких разделителей, если же их нужно много, то можно использовать `<pre>` или свойство `white-space`.

```html
Mozilla<br />
331 E. Evelyn Avenue<br />
Mountain View, CA<br />
94041<br />
USA<br />
```

![[Структура_веб-страницы_br.png]]

### `<hr>` Разделитель темы (горизонтальная линия)

#### Предназначение

Элемент `<hr>` представляет собой тематический разрыв между элементами уровня абзаца, к примеру, смену сцены в статье или сдвиг темы внутри раздела.

Сейчас этот элемент представляется в семантических целей, если нужно просто горизонтальная линия, то для этого можно использовать `CSS`.
#### Атрибуты

Элемент принимает только глобальные атрибуты.
#### Пример

```html
<p>
  This is the first paragraph of text. This is the first paragraph of text. This
  is the first paragraph of text. This is the first paragraph of text.
</p>

<hr />

<p>
  This is the second paragraph of text. This is the second paragraph of text.
  This is the second paragraph of text. This is the second paragraph of text.
</p>
```

![[Структура_веб-страницы_hr.png]]
